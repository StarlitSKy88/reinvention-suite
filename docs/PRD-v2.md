## 1. 产品一句话定位

**再出发 = 35+ 失业群体免费使用的再就业助手，由各地人社局再就业服务采购支持。**

> "让 35+ 不必再假装 25" — 全网匹配企业，主动协助弥补能力差距，让再就业周期从 8 个月缩短到 4 个月。

---

## 2. 目标用户画像

### 核心用户：35-50 岁失业群体

```
人口规模     ：8000 万存量（2026 中国）
典型用户     ：42 岁，互联网/制造业/金融，失业 6-12 个月
核心痛点     ：年龄歧视 + 简历被秒筛 + 不知道差什么
决策周期     ：短（急需现金流）
付费能力     ：低（房贷/家庭支出）
使用场景     ：手机为主（60%+），浏览器/小程序
```

### 次要用户：转型群体

- **应届/初级**（占比 15%）：不主推，但可复用工具
- **50+ 高管**（占比 5%）：客单价高，可作未来扩展

### 不服务用户

- **应届毕业生**：被 51job/Boss 应届版块垄断
- **猎头群体**：商业模型冲突
- **企业 HR**：我们的产品是 to candidate

---

## 3. 商业模式：To-G（政府采购）

### 核心逻辑

```
┌──────────────────────────────────────────┐
│  C 端：完全免费（35+ 失业群体）          │
│  B 端：政府采购（人社局再就业服务）      │
│  核心：C 端获客 + 数据积累 → 政府买单   │
└──────────────────────────────────────────┘
```

### 收入结构

| 阶段 | 单价 | 数量 | 年收入 |
|---|---|---|---|
| 单城市试点 | ¥100w-300w/年 | 1 个 | ¥100w-300w |
| 5 城市复制 | ¥200w-500w/年 | 5 个 | ¥1000w-2500w |
| 30 城市铺开 | ¥400w-1000w/年 | 30 个 | **¥1.2 亿+** |

### 为什么政府愿意买单

1. **KPI 压力**：稳就业是国务院重点，35+ 再就业率是考核指标
2. **政绩展示**：可包装为"数字化再就业示范项目"
3. **解决问题**：35+ 失业是社会稳定隐患，政府缺工具
4. **风险可控**：先免费试点，效果好再采购

## 5. 核心差异化（无竞品）

### 差异化 1：35+ 垂直

- 简历去年龄化（自动隐藏毕业年份、突出近期项目）
- 转型路径建议（从 A 行业到 B 行业，缺什么补什么）
- 年龄友好公司画像（标注哪些公司不歧视年龄）

### 差异化 2：反 AI 痕迹 + 反歧视触发器

- 不止简历优化，深度检测 HR 心理学偏见
- ChatGPT 反而暴露年龄（"资深感"措辞），我们专门解决

### 差异化 3：真实人设工程（反幻觉三重保险）

- **事实库 + 约束 Prompt + 反向校验**
- AI 只翻译不编造
- 卖点：*"No fabrication. No slop. Just better translation of your real wins."*

### 差异化 4：项目孵化器（最核心）

- 10+ 真实可执行的项目模板（开源/写作/MVP/数据实验/...）
- 协助用户做真实项目，不是 AI 编造
- 完成后生成可写进简历的项目描述

## 6. 技术架构

### 整体架构图

```
┌──────────────────────────────────────────────────────────┐
│              Frontend (Next.js 14 App Router)            │
│  C 端：/  简历上传/匹配/改写/项目孵化/投递导航           │
│  B 端：/gov-dashboard  政府专属看板                       │
├──────────────────────────────────────────────────────────┤
│              API Layer (Next.js Route Handlers)           │
│  /api/match /api/rewrite /api/analytics /api/gov        │
├──────────────────────────────────────────────────────────┤
│              AI Layer (Provider 抽象层)                  │
│  MiniMax-M3（主） / Claude Sonnet 4.6（高质量）         │
│  DeepSeek-V3（兜底）                                     │
├──────────────────────────────────────────────────────────┤
│              Python Scraper 微服务                       │
│  Tier 1：Exa/Tavily + Scrapling Fetcher（公司官网）     │
│  Tier 2：Scrapling StealthyFetcher（用户授权爬虫）      │
│  多源调度 + 自动熔断 + 速率限制                          │
├──────────────────────────────────────────────────────────┤
│              Data Layer                                  │
│  客户端：IndexedDB（Dexie.js）— 简历原文本地存           │
│  服务端：PostgreSQL — 用户元数据 + 行为日志（脱敏）      │
│  缓存：Redis                                              │
└──────────────────────────────────────────────────────────┘
```

## 9. 关键风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| **爬虫法律风险** | 平台起诉、IP 封禁 | 用户授权 + 自动熔断 + 数据脱敏 |
| **政府 BD 周期长** | 现金流压力 | 试点免费 + 多渠道并行 |
| **效果不确定** | 政府不续约 | 免费试点 → 数据说话 → 续约 |
| **35+ 用户不信任** | 转化率低 | 隐私保护 + 政府背书 + 数据透明 |
| **AI 成本爆炸** | 亏损 | 缓存 + DeepSeek 兜底 + 限额 |
| **政策变化** | 业务受损 | 稳就业是长期国策，5-10 年内稳定 |
