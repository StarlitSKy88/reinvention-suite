## 📅 12 个月路线图

### Phase 1：基础资质（M1-M2）

**目标**：拿到营业执照 + 人力资源服务许可证

#### 1.1 营业执照（含人力资源服务）

**时间**：1-2 周
**成本**：¥0
**流程**：
1. 公司核名（建议 3-5 个备用名）
2. 准备注册材料（股东、监事、注册资金、注册地址）
3. 提交工商注册（线上 e 窗通）
4. 领取营业执照
5. 刻章 + 银行开户 + 税务报到

**注意事项**：
- 经营范围必须包含"人力资源服务"、"互联网信息服务"、"软件开发"
- 注册资金建议 ¥1000w（政府项目信任度）
- 注册地址可以是虚拟地址（但建议实际地址）

#### 1.2 人力资源服务许可证

**时间**：1-3 月
**成本**：¥1w-5w
**流程**：
1. 准备申请材料：
   - 营业执照副本
   - 法定代表人身份证
   - 经营场所证明
   - 3 名以上从业人员资格证书（HR 资格证）
   - 公司章程
   - 信息系统安全等级保护备案证明
2. 提交区/县人力资源和社会保障局
3. 现场核查（场地、设施、人员）
4. 颁发《人力资源服务许可证》

**注意事项**：
- 3 名从业人员需持证（可以招聘时有证的人）
- 部分地区要求实际办公场地（不接收虚拟地址）
- 提前与当地人社局沟通，了解具体要求

#### 1.3 增值电信业务经营许可证（ICP）

**时间**：1-3 月
**成本**：¥1w-3w
**流程**：
1. 准备申请材料：
   - 公司营业执照
   - 法定代表人身份证
   - 域名证书
   - 服务器放置地证明
2. 工信部 ICP/IP 地址/域名信息备案管理系统
3. 提交省通信管理局
4. 现场核查（部分省份）
5. 颁发 ICP 许可证

**注意事项**：
- 必须先有备案号（ICP 备案）
- 服务器必须放在中国大陆（不能放境外）
- 部分业务类型需要前置审批（如新闻、出版）

### Phase 2：数据安全资质（M3-M6）

**目标**：拿到等保 2.0 三级认证 + 隐私信息管理体系认证

#### 2.1 等保 2.0 三级认证（关键）

**时间**：3-6 月
**成本**：¥10w-30w
**意义**：政府项目最看重的资质之一

**流程**：
1. **定级备案**（1 月）
   - 系统定级（三级）
   - 填写《信息系统安全等级保护备案表》
   - 提交公安机关

2. **建设整改**（2-4 月）
   - 物理安全：机房、门禁、监控
   - 网络安全：防火墙、入侵检测
   - 主机安全：操作系统加固
   - 应用安全：身份认证、访问控制
   - 数据安全：加密、备份、脱敏
   - 安全管理：制度、人员、运维

3. **测评**（1-2 月）
   - 选择等保测评机构（公安部推荐名单）
   - 现场测评（技术 + 管理）
   - 整改不符合项
   - 出具测评报告

4. **备案**（1 周）
   - 提交测评报告
   - 公安机关颁发备案证明

**关键点**：
- 三级系统每年至少 1 次测评
- 找有经验的测评机构（推荐：等保测评头部公司）
- 配合整改需要技术 + 管理双管齐下

#### 2.2 隐私信息管理体系认证（推荐）

**时间**：3-6 月
**成本**：¥5w-15w
**意义**：体现对用户隐私的重视

**流程**：
1. 建立隐私信息管理体系（PIMS）
2. 准备文件（隐私政策、数据流转图、风险评估）
3. 选择认证机构
4. 第一阶段审核（文件审核）
5. 第二阶段审核（现场审核）
6. 颁发证书（有效期 3 年）

**关键点**：
- 与等保同步进行，节约时间
- 用户协议、隐私政策需要完善
- 数据流转需要文档化

### Phase 3：高级资质（M7-M12）

**目标**：拿到 ISO 27001 + ISO 27701 + 国密算法改造（视情况）

#### 3.1 ISO 27001（加分）

**时间**：6-12 月
**成本**：¥10w-30w
**意义**：国际通用安全认证

**流程**：
1. 建立信息安全管理体系（ISMS）
2. 风险评估 + 处理
3. 文件准备
4. 内部审核 + 管理评审
5. 认证机构审核（一阶段 + 二阶段）
6. 颁发证书（有效期 3 年）

#### 3.2 ISO 27701（隐私扩展）

**时间**：6-12 月
**成本**：¥10w-20w
**意义**：ISO 27001 的隐私扩展

#### 3.3 国密算法改造（部分政府要求）

**时间**：6-12 月
**成本**：¥20w-50w
**意义**：部分政府项目要求国密算法（SM2/SM3/SM4）

**改造内容**：
- HTTPS 证书国密化
- 数据库加密国密化
- 加密机部署
- 应用层国密 API 集成

## ⚠️ 关键注意事项

1. **资质是前置条件**
   - 部分项目要求投标前必须具备资质
   - 提前 3-6 个月开始准备

2. **持续合规**
   - 等保每年复测
   - ISO 每 3 年续证
   - 不要"一锤子买卖"

3. **数据安全是核心**
   - 政府最关心数据安全
   - 等保 + 隐私认证是政府信任的基石

4. **本地化要求**
   - 部分省份要求服务器本地化
   - 部分项目要求数据不出省
   - 提前规划多区域部署

5. **持续投入**
   - 合规是持续投入，不是一次性
   - 预留每年 ¥20w-30w 合规预算
